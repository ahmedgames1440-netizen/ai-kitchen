# 🌯🤖 مطعم المستقبل – AI Kitchen

لعبة إدارة مطعم عربية/خليجية بطابع كرتوني — مبنية من ملف تصميم اللعبة (GDD) الإصدار 1.0، مع تنفيذ كامل للنموذج الأولي وأنظمة الذكاء الاصطناعي الثلاثة.

**🎮 العب الآن:** https://ahmedgames1440-netizen.github.io/ai-kitchen/

- لعبة ويب خفيفة تعمل على الجوال والتابلت والكمبيوتر (RTL بالكامل).
- بدون سيرفر — كل أنظمة الذكاء الاصطناعي تعمل محلياً في المتصفح.
- التقدم يُحفظ تلقائياً في جهازك (localStorage).

## 💡 اقتراحاتي المضافة (تحسينات على الـ GDD)

1. **عملة مزدوجة بمعنى:** الذهب 🪙 لا يُشترى — يُكسب فقط من **التسليم المثالي** (صبر > 75%)، وهو مفتاح مطبخ الذكاء الاصطناعي. هذا يربط المهارة بالمحتوى الإبداعي بدل الشراء العشوائي.
2. **المحادثة كآلية لعب لا زينة:** الرد الصحيح يرفع صبر الزبون فعلياً (شريان حياة وقت الزحمة)، والرد السيئ يعاقبك — والرسائل مخصصة حسب شخصية الزبون.
3. **الناقد 🧐 سلاح ذو حدين:** تأثيره على التقييم مضاعف (سلباً وإيجاباً) ويعطي ذهباً مضاعفاً — يخلق قرار أولويات حقيقي.
4. **الصينية كمورد محدود:** لا يمكن الطبخ إذا امتلأت (الطبخ الجاري يحجز مكاناً) — يمنع التخزين اللانهائي ويضيف إدارة موارد.
5. **تقرير يتذكر:** يحتفظ بآخر 14 يوماً ويقارن أرباحك بالأمس ويحدد نوع الزبون الذي يزعل عندك أكثر.
6. **أطباق الذكاء الاصطناعي أغلى وأبطأ:** سعر أعلى (11–18) مقابل وقت طبخ أطول — مخاطرة/مكافأة بدل كونها مجرد سكين (بحد أقصى 4 أطباق مبتكرة كي لا تتضخم القائمة).
7. **جلسات قصيرة فعلاً:** اليوم يبدأ من ~72 ثانية ويطول تدريجياً (سقف 150 ث) — يناسب لاعب الجوال كما حدد الـ GDD.
8. **إيقاف لطيف:** زر "إنهاء اليوم" في الإعدادات بدل خسارة التقدم عند الخروج.

## 🎭 تحديث 2: الشخصيات المميزة وترقية اللعب

**شخصيات مميزة (VIP)** برسوم SVG كرتونية، تزور مطعمك من اليوم الثاني ولكل واحدة ميكانيكية خاصة:

| الشخصية | القدرة |
|---|---|
| 📿 **شيخ يوسف** | صبور جداً — رضاه "بركة" ترفع تقييمك بقوة (تأثيره ×3) |
| 🧒 **خليل** | طفل بطلب صغير — إذا رضي بسرعة يجيب ربعه (موجة زبائن فورية) |
| 🕵️ **أحمد الفهد** | مفتش الجودة — تسليم مثالي = 3 ذهب، وإغضابه كارثة على التقييم |
| 👓 **العم سالم** | ينسى ويغيّر طلبه في منتصف الانتظار! |
| 🎩 **أبو سمير** | تاجر يدفع **ضعف السعر** ويطلب 3–4 أصناف |

**ترقيات طريقة اللعب:**
- **تسليم بلمسة واحدة:** اضغط الزبون مباشرة ويُسلَّم أول صنف مناسب من الصينية تلقائياً (التحديد اليدوي ما زال متاحاً للتحكم الدقيق).
- **نظام الكومبو 🔥:** تسليمات متتالية (بصبر ≥50%) ترفع مضاعف الأرباح حتى ×1.8 — الخطأ أو الزبون الغاضب يكسره.
- **ساعة الذروة ⏰:** من اليوم الثالث، 15 ثانية زحمة مضاعفة وأرباح ×1.5 بتوقيت عشوائي.
- **الهدف اليومي 🎯:** ظاهر في الشريط العلوي — تحقيقه يمنح +20% مكافأة و3 ذهب.
- **سلة مهملات 🗑️:** ارمِ الصنف الخاطئ بدل ما يحجز الصينية.

## 🧊 تحديث 3: المطعم صار ثلاثي الأبعاد!

مشهد WebGL كامل عبر **Three.js**:

- **مطعم 3D:** أرضية خشبية، جدار بنوافذ نيون، لوحة مضيئة، كاونتر، سيخ شاورما يدور في الخلفية، وإضاءات معلقة.
- **شخصيات مجسمة low-poly مبنية بالكود** (بدون أي ملفات نماذج): الغترة والعقال، الشماغ الأحمر، الطربوش والشنب، لحية شيخ يوسف، نظارات العم سالم، وخليل الطفل الصغير — كل التفاصيل geometry حقيقية.
- **حركة حية:** الزبون يمشي لمكانه، يتمايل وهو واقف، يهتز ويحمرّ وجهه لما يقارب يفقد صبره، يقفز خارجاً سعيداً أو يخرج غاضباً بدوران.
- **التفاعل بالنقر على المجسم نفسه** (raycasting) أو على فقاعته — فقاعات الطلبات والأسماء وأشرطة الصبر طبقة HTML تتبع رؤوس المجسمات بدقة كل إطار.
- **وضع 2D احتياطي تلقائي:** إذا ما توفر WebGL أو ما تحمّلت المكتبة، ترجع اللعبة لبطاقات 2D بدون أي كسر.

## 🔮 خارطة الطريق (مقترحات للمرحلة القادمة)

- **ربط LLM حقيقي:** استبدال مولّد الأطباق والمحادثة المحلي بـ Claude API عبر سيرفر وسيط صغير (FastAPI) — البنية جاهزة: `aiGenerateDish()` و`CHAT_EVENTS` نقاط استبدال واضحة.
- **توليد صور الأطباق:** Stable Diffusion / صور بدل الإيموجي.
- **حفظ سحابي ولوحة صدارة** بين اللاعبين.
- **نسخة Flutter/متجر** بعد إثبات المتعة في نسخة الويب (نفس نهج مشروع موجة).
- **أصوات وموسيقى كاملة** بدل النغمات المولدة.

## 🛠️ التقنية

- HTML + CSS + JavaScript — بدون خطوة بناء. المكتبة الوحيدة: Three.js (CDN) للمشهد ثلاثي الأبعاد.
- `game.js`: منطق اللعبة كاملاً (زبائن، طبخ، محادثة، مولد أطباق، تحليل، متجر، حفظ).
